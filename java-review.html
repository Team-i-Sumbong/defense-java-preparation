Module 1

Java Foundations: Variables, Types, and Syntax

Concept

Variables store values that the program needs while it runs.

Java uses data types such as int, double, boolean, and String so the compiler knows what kind of value is being stored.

Why It Matters

  • Correct types reduce logic mistakes.
  • They make method contracts easier to understand.
  • They improve readability during code review and defense.

Module 2

Control Flow: Conditionals and Loops

Concept

Conditionals decide which path the program should follow.

Statements like if, else, and switch allow Java programs to respond to different inputs and states.

Connection

In login logic, the program checks whether an account exists, whether the password matches, and whether the account is verified before allowing access.

Defense line: Java control flow lets the system make decisions based on rules, such as whether a user is verified or whether required report fields are complete.

Module 3

Classes and Objects: The Core of Java OOP

Concept

A class defines structure and behavior; an object is one usable instance.

This is one of the most important review areas because i-Sumbong is built from classes that represent users, reports, services, and repositories.

Project Evidence

Class Role
User Represents a system user
IssueReport Represents a submitted concern
AuthService Represents business-rule behavior for accounts

Module 4

OOP Principles: Abstraction, Inheritance, Polymorphism, Encapsulation

Abstraction

Abstraction means focusing on the essential role of a component without exposing every low-level detail.

In defense, abstraction helps you explain what a class is for before discussing its implementation details.

Module 5

Methods, Constructors, and Encapsulation

Methods

A method defines behavior. It may receive parameters, process data, and return a result. Good methods have clear responsibilities and names that match their purpose.

Constructors

Constructors help initialize objects when they are created. They are especially useful when an object should start with meaningful values instead of empty state.

Defense line: In Java, methods define behavior, while constructors define how an object begins its life with valid starting data.
