Module 1

Variables and Data Types

Key Idea

Variables store values, and data types tell Java what kind of value each variable holds.

This matters because the compiler and the programmer both need to know what kind of data is expected.

Project Example

In i-Sumbong, fields such as report titles use String, flags can use boolean, and counters or identifiers may use numeric types.

Defense Line

We use variables and data types to store information in a controlled form, which helps the program process the correct kind of data consistently.

Module 2

Control Flow

Key Idea

Control flow lets the program choose what happens next based on conditions.

The most common defense example is the use of if statements in validation and login checks.

Project Example

During login, the system checks whether the account exists, whether the password matches, and whether the user is allowed to continue.

Defense Line

Java control flow helps the system make rule-based decisions, such as whether a login should succeed or whether report input is valid.

Module 3

Classes and Objects

Key Idea

A class is a blueprint, while an object is one actual instance created from it.

This idea is central because the project is built from model, service, repository, and view classes.

Project Example

Class Role
User Represents a person in the system
IssueReport Represents one submitted concern
AuthService Performs account-related behavior

Defense Line

We use classes to organize data and behavior into meaningful components, and objects let the system work with real instances of those components.

Module 4

Methods and Parameters

Key Idea

Methods define behavior, and parameters show what input that behavior needs.

Good method explanations start with purpose, then mention input and output.

Project Example

public boolean registerResident(User user)

This method receives a user object, performs checks, saves the data, and returns whether the action succeeded.

Defense Line

Methods keep behavior organized, while parameters make the required input explicit so each operation is easier to read and explain.

Module 5

Encapsulation and One Inheritance Example

Encapsulation

Encapsulation means object data is accessed through controlled methods instead of direct public access.

This keeps state more organized and makes future validation easier to add at the access points.

Open support note

Encapsulation example: Getters and setters control how fields in IssueReport or User are read and changed.

Inheritance example: App extends JavaFX Application, which shows reuse of framework behavior.
